## 3) What Each Top-Level Folder Is For

### `casecast/`
Workspace root. Contains:
- A minimal root `package.json` (some dependencies listed at workspace level)
- `court-ai-frontend/` (the real runnable web app)

Why this exists:
- Sometimes developers keep shared tooling/dependencies in root.
- In your setup, frontend app scripts are in `court-ai-frontend/package.json`.

### `court-ai-frontend/`
Actual React web app created with Vite.

### `court-ai-frontend/src/`
Main source code for React components and CSS.

### `court-ai-frontend/public/`
Static files copied as-is to build output.
- `loader.mp4` is your landing background video.
- Anything in `public` is referenced with absolute path like `/loader.mp4`.

### `court-ai-frontend/dist/`
Production build output generated by `npm run build`.

### `node_modules/`
Installed dependencies (auto-generated). Not edited manually.

## 5) Where Tailwind and CSS Are, and How They Work Here

Tailwind setup is present in three places:
1. `tailwind.config.js` - scan paths
2. `postcss.config.js` - plugin hookup
3. `src/index.css` - `@tailwind` directives

How it works in your current code:
- Tailwind is installed and configured.
- Current main landing page styling is mostly custom CSS classes in `index.css`.
- `src/pages/Landing.jsx` still uses Tailwind utility classes, but that page is not currently mounted.

## 6) Main React Flow (Important)

1. Browser loads `index.html`.
2. `index.html` loads `src/main.jsx`.
3. `main.jsx` mounts `<App />` into `#root`.
4. `App.jsx` renders full landing UI:
   - video
   - fade overlay
   - cursor letters
   - navbar
   - hero content
5. `index.css` styles every class used in `App.jsx`.

So the central React part is:
- `src/main.jsx` (entry bootstrap)
- `src/App.jsx` (main UI component)

## 7) What Each Landing Class/Div Does (App.jsx + index.css)

### Container and Background

- `landing-shell`
  - Full-screen container (`100dvh`), positioning context, and mouse event surface.
- `bg-video`
  - Absolute full-screen video layer behind all content.
- `entry-fade`
  - Black overlay used for initial page fade-in.
  - Removed by adding `is-loaded` class when video is ready.
- `landing-shell is-loaded`
  - State class toggled by React when video can play.

### Cursor Letter Effect

- `cursor-letter-cluster`
  - A fixed layer anchored to current mouse position via CSS vars.
- `cursor-letter-cluster is-active`
  - Makes cluster visible while cursor is inside landing shell.
- `cluster-letter`
  - Individual animated letters that orbit/float around cursor.
  - Styled using CSS variables passed from React (`--letter-dx`, `--letter-dy`, etc).

### Navbar

- `landing-nav-wrap`
  - Positions navbar region above content.
- `landing-nav`
  - Main navbar container (glass-like white background, rounded pill shape).
- `brand-mark`
  - CASECAST left brand chip.
- `nav-links`
  - Horizontal navigation list container.
- `nav-cta`
  - Right-side CTA button (Request Demo).

### Hero Section

- `hero-wrap`
  - Main content vertical area below navbar.
- `hero-content`
  - Centered content block (kicker/title/subtitle/actions).
- `hero-kicker`
  - Small uppercase intro text above title.
- `dot-title`
  - Main CASE CAST title using Doto with variation setting `ROND`.
- `hero-subtitle`
  - Description box below title.
- `hero-actions`
  - Row for action links/buttons.
- `text-link`
  - Remaining action link (Explore Platform).

## 9) Why This Architecture Works

- Vite gives fast development and optimized production builds.
- React keeps UI behavior and state logic organized in components.
- Framer Motion handles smooth entrance animations.
- Custom CSS gives you full control over aesthetic details.
- Tailwind is available for utility-first styling if you decide to use it more.
